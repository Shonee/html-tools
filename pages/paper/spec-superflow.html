spec-superflow

源码级融合 OpenSpec 规划引擎 + Superpowers 执行纪律的 AI 编程工作流插件

TypeScript v0.8.4 9 Skills 7 平台 ⭐ 195

解决什么问题

AI 编程两大失控点：还没想清楚就开始写代码，以及规划写完但执行跑偏。spec-superflow 用执行契约把规划和实现焊死，确保「想清楚再做、做了不跑偏」。

核心特性

●
Spec First— 没有稳定规划工件，不允许进入实现
●
执行契约桥接— execution-contract.md 是规划到实现的唯一交接层
●
TDD + SDD— 子代理驱动 + 双层审查，No test = No code
●
Schema 验证引擎— 规划工件经引擎验证，不合格不允许进入桥接
●
快速路径— hotfix/tweak 自动模式检测，小变更不被流程拖慢
●
自包含— 不需要安装 OpenSpec 或 Superpowers 运行时

技术亮点

01

内容级状态检测

不只检查文件存在，还比较 proposal 范围 vs 契约意图锁

02

Schema 验证引擎

TypeScript 引擎验证 SHALL/MUST 语义、Delta 冲突、中英文分词

03

SDD 子代理驱动

派实施子代理 → 生成 diff → 派审查子代理 → 双向判断

04

多平台 phase-guard

ssf inject 生成 Claude/Cursor/Copilot/Gemini 的阶段守护产物
