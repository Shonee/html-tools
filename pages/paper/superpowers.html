解决什么问题

AI 编码 Agent 能力强但缺纪律——不问需求就动手、跳过测试、猜测修 Bug。Superpowers 用「铁律」和「硬门」机制，强制 Agent 按工程规范行事：先设计、先写测试、先找根因。

# 无需额外操作，Session 启动自动激活正常和 AI 对话即可Agent 会主动问需求细节而非直接写码

🔧 常见问题

Skills 不自动触发 → 确认插件已安装并重启 Agent 会话

响应变慢 → 正常现象（多 Skill prompt 增加），简单任务可说「跳过设计」

Windows .sh 弹窗 → 使用 WSL 或忽略（不影响核心功能）

确认是否生效 → Agent 是否在写代码前先问你需求

技术亮点

01

Iron Law 铁律机制

命令式约束（非建议）：违反则强制回滚重来

02

Subagent 隔离架构

每任务独立 Agent + 独立上下文，防止长会话退化

03

零代码 Skill 定义

纯 Markdown SKILL.md 文件，任何 LLM 可解析执行

04

Visual Brainstorming

v5+ 支持浏览器内交互 HTML 原型，替代 ASCII 图

优势与局限
