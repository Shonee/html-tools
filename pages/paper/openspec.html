核心特性

●
Explore 探索— 无风险思考伙伴，先想清楚再动手
●
Propose 提案— 一键生成 proposal + specs + design + tasks
●
Apply 执行— 按 tasks 逐项实施，每步可审可断
●
Archive 归档— 完成后归档，规范自动累积为知识库
●
Delta Spec— 只记录变化部分，轻量增量式管理
●
Stores (Beta)— 跨项目、跨团队共享规范

完整使用指南

1 安装 OpenSpec

# 需要 Node.js >= 20.19.0npm install -g @fission-ai/openspec@latest# 也支持 pnpm / yarn / bun / nixpnpm add -g @fission-ai/openspec@latest

全局安装 CLI 工具

2 初始化项目

cd your-project# 生成 openspec/ 目录结构openspec init

选择 AI 工具 → 自动生成 project.md 和 AGENTS.md

3 开始使用（在 AI 工具内）

/opsx:explore/opsx:propose add-feature

AI 自动读取 openspec/ 下的指令文件并响应斜杠命令

4 日常开发流程

# 探索 → 提案 → 执行 → 归档/opsx:explore# 审核提案后执行/opsx:apply/opsx:archive

5 更新与卸载

# 升级到最新版npm install -g @fission-ai/openspec@latestopenspec update# 卸载npm uninstall -g @fission-ai/openspecrm -rf openspec/

⚙ Profile 配置

# 切换工作流 Profileopenspec config profile# 刷新 AI 指令openspec update

🔧 常见问题

命令不显示 → openspec update 刷新指令文件

Node 版本不够 → nvm install 20 升级运行时

工具不支持 → 查看 Supported Tools 文档确认

遥测关闭 → export OPENSPEC_TELEMETRY=0

技术亮点

01

Delta Spec 增量规范

只记录变化部分，归档后自动合并到 source-of-truth 文档

02

无阶段门设计

任何 artifact 随时可修改，不存在"锁定后不可变"的刚性限制

03

25+ 工具适配器

通过 AGENTS.md / slash commands 自动适配不同 AI 编码工具

04

Community Schema 生态

第三方工作流扩展，类似插件市场，自定义模板和规则
